# NSMP Sources Sync

Набор модулей для git-подобныой работы с исходниками NSMP.

<u>Не является официальным инструментом компании Naumen</u>. Naumen имеет свои инструменты для работы с исходниками (и в целом
довольно большой объем интересных библиотек), за ними обращайтесь в техподдержку Naumen.

## Модули

| Gradle project | Папка           | Назначение                                                                                                                                                                                      | Maven artifactId | README                          |
|----------------|-----------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------|---------------------------------|
| `:core`        | `module_core`   | Библиотека, пригодная к использованию в любых приложения (пишите свои плагины, если хотите)                                                                                                     | `core`           | [тык](/module_core/README.md)   | 
| `:cli`         | `module_cli`    | Консольная утилита, обеспечивает доступ к библиотеке                                                                                                                                            | `cli`            | [тык](/module_cli/README.md)    |
| `:gradle`      | `module_gradle` | Gradle-плагин. Та же консольная утилита, но работает в экосистеме gradle, то есть может встраиваться в процессы сборки. Был написан случайно и не факт что будет долго поддерживать, посмотрим. | `gradle`         | [тык](/module_gradle/README.md) |

Общая идеология модулей заключается в наличии в них трех команд:

- `pull` - получить определенные исходники из инсталляции
- `syncCheck` - проверить, производилось ли изменение исходника в инсталляции с момента последнего pull
- `push` - загрузить исходники на инсталляцию, по умолчанию перед загрузкой выполняется `syncCheck`, который не
  позволяет выполнить загрузку, если исходник был модифицирован с момента последнего `pull`. Параметр force отключается
  проверку.

Общие координаты:

- `group`: `ru.kazantsev.nsmp.sdk.sources_sync`

## Общие требования

- JDK 21
- Gradle Wrapper из репозитория

## Важная информация

При работе с исходниками с использованием библиотеки важно сохранять уникальность наименования исходника в рамках source set.
Так как в NSMP не запрещается создавать скрипт с кодом (идентификатором скрипта), который дублирует код модуля,
для работы библиотеки требуются нестандартные source sets, так как работа в рамках одного source set невозможна, а по умолчанию он всегда один.

Библиотека самостоятельно папки с наборами исходников:

- `src/main/scripts` - для скриптов
- `src/main/modules` - для модулей
- `src/main/advimports` - для импортов

Для адекватного восприятия исходников в этих папках вашей IDE, вам требуется самостоятельно подключить данные папки как
source sets в вашу систему сборки (кроме gradle плагина, он делает это при подключении).

Для хранения метаинформации библиотека так же создаст папку `.nsmp_sdk` в вашем проекте.
